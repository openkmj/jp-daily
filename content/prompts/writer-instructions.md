# Writer Instructions — jp-daily 월별 콘텐츠 생성

## 목적

한국어 화자(왕초보)를 위한 일본어 일일 학습 데이터 한 달치 생성.

## 입력 자료 (반드시 먼저 읽을 것)

1. `content/curriculum.yaml` — 8개월 커리큘럼 전체. **너의 월에 해당하는 섹션만** 따른다.
2. `scripts/sample-2026-04.json` — 4월 7일치 시드. 출력 schema/스타일/토큰화 방식의 ground truth.

## Schema (출력 파일 형태)

`content/drafts/YYYY-MM.json` 위치에 다음 구조로 저장:

```json
{
  "version": 1,
  "month": "2026-05",
  "lessons": {
    "2026-05-01": {
      "date": "2026-05-01",
      "sentences": [
        {
          "tokens": [
            { "jp": "おはよう", "pron": "오하요-", "meaning": "안녕(아침)" }
          ],
          "meaning": "안녕하세요 (아침 인사).",
          "keyPoints": ["선택사항: 0~3개 한국어 짧은 문법/용법 설명"]
        },
        { ... 두 번째 문장 },
        { ... 세 번째 문장 }
      ]
    },
    "2026-05-02": { ... },
    ...
  }
}
```

## 강제 규칙

- 그 월의 **모든 날짜**를 채운다 (1일~말일까지 빠짐 없이).
- 하루는 **정확히 sentences 3개**.
- 각 sentence:
  - `tokens` 배열: 일본어 문장을 자연스러운 단위로 분해
    - 조사(は/を/が/に/で/と/から/まで/の/も)는 **별도 토큰**
    - 동사 활용형은 한 토큰 (예: `飲みます`, `見せて`, `行きました`는 통째로)
    - 외래어 전체가 한 토큰 (예: `コーヒー`, `カード`)
    - 한자 + 가나 복합어는 자연 단위 (예: `お願いします`, `ありがとう`)
  - 각 token:
    - `jp`: 문장에 등장한 그대로의 표기 (한자/가나/카타카나)
    - `pron`: **한글로만** 쓴 발음 (한자/일본어 글자 절대 X). 장음은 `-` 사용. 예: `쿄-`, `이마`, `오네가이시마스`.
    - `meaning`: 매우 짧은 한국어 의미 (1~5음절 권장). 동사는 `~다` 또는 활용형 표시.
  - `meaning`: 문장 전체의 자연스러운 한국어 번역
  - `keyPoints`: 0~3개. 그 문장에서만 새로 등장한 문법/용법을 짧게 설명. 매일 안 넣어도 됨. 한 주에 2~4개 정도면 적정.
- 토큰들의 `jp`를 공백 없이 이어붙이면 원문과 (구두점 제외) 일치해야 한다.
- 모든 한국어 텍스트는 **한국어로만** (영어 X). 단어 수준 영어 외래어 OK ("PC", "TV" 등 아주 일반적인 것).
- 발음 표기 일관성:
  - は (조사) = `와`, へ (조사) = `에`, を = `오`
  - 장음 = `-` (예: コーヒー = `코-히-`, おはよう = `오하요-`)
  - 촉음 = 작은 `ㅅ`을 받침으로 (예: ちょっと = `춋토`, 楽しかった = `타노시캇타`)
  - 발음 표기는 한국인이 읽었을 때 일본어처럼 들리는 것 우선
- 문장은 **실제 회화에서 쓰일 만한 자연스러운 표현**. 책에만 나오는 어색한 문장 X.

## 커리큘럼 따르기

- `content/curriculum.yaml`의 너의 월 섹션:
  - `weeks`: 주별 토픽. 그 주의 모든 날짜는 토픽 안에서 변주
  - `seed`: 핵심 어휘 힌트. 반드시 모두 사용은 아니나, 비슷한 결의 단어들로 채울 것
  - `grammar_focus`: 그 월에 새로 노출하는 문법 패턴들. 적절히 분배해서 등장시킴
  - `new_vocab_target`: 신규 단어 대략 갯수 (±10 OK)
- 4월에 이미 등장한 단어(아래 목록)는 자유롭게 재사용. 익숙해지게 만드는 게 학습 효과.

### 4월 누적 단어 (자유 재사용)
```
はじめまして, 私, は, ミンジェ, です, よろしく, お願いします,
メニュー, を, 見せて, ください, これ, とても, 美味しい,
駅, どこ, ですか, ここ, から, 近い, ありがとう, ございます,
今, 何時, 三時, に, 会いましょう, ちょっと, 遅れます,
いくら, 高い, ね, カード, で, 払えます, か,
今日, 楽しかった, 疲れました, また,
おはよう, いい, 天気, コーヒー, 飲みます
```

## 톤 / 스타일

- 정중체(`~です/ます`) 기본. 친근한 종조사(`ね`, `よ`) 가끔.
- 너무 격식 차린 비즈니스 표현은 N4 이후로 미룸 (12월 일부에서 살짝 허용).
- 일상 회화 위주. 글말체(です/ますだけど・・・) 표현은 신중히.
- 한 주 안에서 sentences는 가능하면 같은 상황의 흐름으로. 각 날짜는 독립적이지만 주제는 일관.

## 검증 (저장 전 self-check)

저장 직전 다음을 모두 확인:
1. `month` 필드 == 너의 월
2. `lessons` 키 = 그 월의 모든 날짜 ("2026-05-01" ~ "2026-05-31")
3. 각 day = sentences 길이 3
4. 모든 token에 jp/pron/meaning 다 채워짐
5. token jp 합치기 = 원문 (구두점 다음에 띄어쓰기 같은 거 제외)
6. pron에 한자/가나 안 섞임
7. meaning에 영어 안 섞임
8. JSON 유효 (인용부호, 콤마 등)

## 분량 가이드

- 한 달치 = 28~31일
- 각 day 평균 토큰 수: 15~25
- 한 달 출력 토큰: 대략 8~12K. 한 번에 끝낼 수 있어야 함.

## 출력

`content/drafts/YYYY-MM.json` 한 파일. JSON.indented (가독성).
